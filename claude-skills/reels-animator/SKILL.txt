## Cos'è questa skill

Sistema completo per produrre reel animati con dati clinici partendo da un video parlato.
Basato su **Remotion** (render React → MP4) + **Whisper** (trascrizione) + **FFmpeg** (transcodifica).
Costo: zero. Nessun abbonamento, nessuna API esterna.

## Workflow per ogni reel

### Step 1 – Prepara il video

Se il video è MOV/HEVC (iPhone):
```bash
bash scripts/transcode.sh C:\Users\umber\Desktop\mio-reel.MOV
```
Il file `.mp4` viene salvato automaticamente in `starter/public/`.

Se è già MP4 H.264, copialo direttamente:
```bash
cp C:\Users\umber\Desktop\mio-reel.mp4 starter/public/
```

### Step 2 – Trascrivi il parlato

```bash
python3 scripts/whisper-transcribe.py starter/public/mio-reel.mp4
```
Output: blocco JSON da incollare nel campo `transcript` del `props.json`.
**Controlla sempre**: nomi brand, numeri, termini clinici (CFT, NDI, HIT-6, CCFT...).

### Step 3 – Scrivi il props.json

Apri `starter/props.json` e modifica:

```json
{
  "videoSrc": "nome-del-tuo-video.mp4",
  "brandHandle": "@studio.mantovan",
  "accentColor": "#1A9EC9",
  "transcript": [ ... ],
  "animations": [ ... ]
}
```

### Step 4 – Anteprima (opzionale ma consigliato)

```bash
npx remotion studio src/index.ts
```
Apre nel browser. Scorri frame per frame per verificare timing.

### Step 5 – Render finale

**Layout B – SplitReelComposition** (consigliato per talking head):
```bash
npm run render:split
```

**Layout A – ReelComposition** (per b-roll o sfondo scuro):
```bash
npm run render:overlay
```

Output in `starter/out/reel-finale.mp4`. Pronto per Instagram.

## Quale layout usare

| Situazione | Layout |
|------------|--------|
| Parli guardando la camera, citi numeri | **B – Split** ✅ (consigliato) |
| B-roll, sfondo clinico, estetica cinematica | **A – Overlay** |
| Presentazione o tutorial con grafici | **B – Split** |
| Hook forte con glitch o quote di impatto | **A – Overlay** |

**Layout B dettagli**: 40% superiore (sfondo ink #1F3A40) per animazioni, 60% inferiore per il video con faccia. Barra teal in cima, handle brand in alto a destra, progress bar in fondo al pannello superiore.

## I 17 tipi di animazione

### TESTO
| Type | Quando usarlo | Esempio clinico |
|------|--------------|-----------------|
| `quote` | Hook iniziale, frase chiave, claim | "Il dolore non significa danno." |
| `typewriter` | Processi, framework, frasi tecniche | "CFT → biopsicosociale → movimento" |
| `text_reveal` | Conclusioni, punti chiave uno alla volta | Tre principi della CFT |
| `glitch_text` | Hook ad alto impatto visivo | "IL RIPOSO NON GUARISCE." |
| `keywords` | Tag del video, concetti chiave | lombalgia, CFT, movimento, evidenza |

### NUMERI
| Type | Quando usarlo | Esempio clinico |
|------|--------------|-----------------|
| `flip_stat` | Il numero più importante del reel | "70%" tasso di miglioramento |
| `number_roll` | KPI secondari, crescita fluida | Da 8 a 2 (NRS dopo trattamento) |

### SEQUENZE
| Type | Quando usarlo | Esempio clinico |
|------|--------------|-----------------|
| `steps` | Tutorial, how-to, protocolli | 3 fasi della CFT |
| `checklist` | Risultati raggiunti, recap | ✅ Valutazione ✅ Piano ✅ Follow-up |
| `countdown` | Conta alla rovescia prima di lista | 3 errori da evitare |

### GRAFICI
| Type | Quando usarlo | Esempio clinico |
|------|--------------|-----------------|
| `chart_bar` | Confronti, serie storiche, outcome | NRS settimana per settimana |
| `chart_line` | Crescita nel tempo | Miglioramento funzionale mese/mese |
| `ring_stat` | Fino a 3 KPI compositi | Engagement + retention + soddisfazione |

### SOCIAL & BRANDING
| Type | Quando usarlo | Esempio clinico |
|------|--------------|-----------------|
| `lower_third` | Presentazione, citazione fonte | "Umberto Mantovan – Fisioterapista" |
| `notification` | Social proof, risultati in tempo reale | "Nuovo paziente: sciatalgia risolta in 8 sett." |
| `badge` | Segnalare passaggio chiave | "STUDIO RESTORE 2023" / "EVIDENCE-BASED" |
| `stat` | Statistica semplice senza animazione | "12 settimane" |

## Regole di timing (CRITICHE)

1. **No overlap**: `startSec + durationSec` deve essere < `startSec` dell'animazione successiva. Gap minimo consigliato: **0.7 secondi**.

2. **Sincronia col parlato**: `startSec` dell'animazione = momento in cui *inizi* a pronunciare quel dato nel video. Non dopo.

3. **Durata minima leggibile**:
   - Animazioni semplici (flip_stat, badge): **5 secondi** minimo
   - Animazioni complesse (ring_stat 3 anelli, chart_bar 4+ barre): **8 secondi** minimo

4. **Ritmo consigliato**: 4-5 animazioni in 30 secondi. Una ogni ~7 secondi. Più di 5 diventa frenetico.

5. **Regola dei numeri**: il valore nel `data` deve corrispondere esattamente al numero che pronunci nel video.

## Troubleshooting

| Problema | Soluzione |
|----------|-----------|
| Video non si carica in preview | Transcodifica con `transcode.sh` |
| Audio mancante nel render | Stessa causa, riconverti in H.264 |
| Animazione che si sovrappone | Controlla gap tra animazioni (>0.7s) |
| Whisper trascrive male i nomi | Correggi a mano nel JSON |
| Render lentissimo | Normale su CPU: ~1-2 min per 30 secondi |
| npm install fallisce | Verifica Node.js 18+: `node --version` |
